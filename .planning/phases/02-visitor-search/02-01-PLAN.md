<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server -> Client | Flag data crosses from server component to client component as props. All flag data is intentionally public. |
| User Input -> Fuse.js | Search query string entered by visitor is passed to Fuse.js search. Client-side only, never sent to server. |
| Client -> DOM | Search results rendered via React JSX. React auto-escapes all string interpolation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Search input rendered in DOM | mitigate | React JSX auto-escapes all interpolated values. Do NOT use dangerouslySetInnerHTML for any search result highlighting. All flag data rendered as text nodes via JSX curly braces. |
| T-02-02 | Information Disclosure | All flags loaded client-side | accept | All flag data (name, row, position) is intentionally public -- this is the core feature. No sensitive fields exist in the Flag type. Acceptable for a memorial event app. |
| T-02-03 | Denial of Service | Extremely long search query | accept | Fuse.js handles arbitrary-length strings gracefully. With ~100 records and client-side execution, no server resources consumed. Worst case: brief UI freeze on very old phones, recoverable by clearing input. |
</threat_model>
